# Contributions

The Contributions section is for **community-created content**. Anyone can submit tutorials, concepts, guides, or deep dives related to Kothar and Aleph. You do not need to be a Kothar employee — if you have experience, insights, or ideas, your work can become part of the official docs.

**Why contribute?**
- Share what you learned building with Kothar or Aleph.
- Help others avoid the pitfalls you already solved.
- Spotlight use cases, patterns, and mental models the core docs don’t cover yet.
- Build your own public reference to point teammates, users, or future you to.

**What kind of content is welcome?**
- Step‑by‑step tutorials and getting‑started walkthroughs.
- Concept explainers that make tricky ideas intuitive.
- Architecture or design notes from real projects.
- Integration guides (tools, frameworks, infra, data pipelines, etc.).
- Opinionated best practices, tradeoffs, and lessons learned.

If you’re not sure where to start, pick a subject you wish existed when you were learning Kothar or Aleph, and write that. Short, focused contributions are absolutely welcome.

**How to submit**
Follow the contribution instructions here:  
<a href="https://github.com/KotharComputing/docs-contributions" target="_blank">https://github.com/KotharComputing/docs-contributions</a>

We review submissions and integrate accepted content into the official documentation. If you care about a topic, that’s the best reason to write about it.
